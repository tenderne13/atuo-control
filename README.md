# auto-control

一个用 Python 写的轻量桌面工具，用来替代按键精灵执行固定窗口的循环点击操作。

## 为什么这样实现

- `Tkinter`：Python 自带，最容易做一个可用的 GUI。
- `ctypes + Windows API`：直接找窗口、恢复窗口、移动窗口，不依赖额外库。
- 系统鼠标事件：可以完成你原来脚本里的 `MoveTo + LeftClick + Delay` 逻辑。

这套方案的优点是部署轻、改配置方便、调试成本低。对你当前这种“固定窗口 + 固定坐标 + 循环执行”的场景来说，比继续维护按键精灵脚本更合适。

## 功能

- 配置目标窗口标题
- 自动恢复并前置窗口
- 自动移动并调整窗口尺寸
- 可视化维护点击步骤
- 支持测试单次和循环执行
- 支持保存 `config.json`
- 内置运行日志

## 运行

```powershell
python app.py
```

首次启动会使用内置默认配置；点击“保存配置”后会在当前目录生成 `config.json`。

## 打包后的程序

已经成功打包出可双击运行的程序：

`dist/auto-control.exe`

说明：

- 双击 `auto-control.exe` 就能直接打开，不需要再手动运行 Python。
- 程序保存的 `config.json` 会放在 exe 所在目录，方便长期使用和迁移。
- 如果你后面把 exe 单独拷到别的目录，配置文件也会跟着在那个目录生成。

## 默认步骤

默认已经预置你提供的三步：

1. 点击开始匹配按钮：`1822, 850`
2. 点击空白区域：`1830, 720`
3. 点击返回按钮：`1290, 940`

每一步默认点击后等待 `10000ms`。

## 注意

- 当前点击坐标是“屏幕绝对坐标”，和你原来的按键精灵脚本保持一致。
- 程序会先把目标窗口移动到 `0, 0` 并调整成 `1920 x 1080`，这样能尽量保证坐标稳定。
- 如果后面你想升级成“相对窗口坐标”或“图像识别后再点击”，这个结构也方便继续扩展。
